  **Sample Implementation Snippet (Transaction Safety):**
  ```python
  # Python/Django ORM: Atomic Transaction for BNPL Ledger
  from django.db import transaction
  from .exceptions import InsufficientCreditError

  def process_bnpl_purchase(user_id, merchant_id, amount):
      with transaction.atomic():
          user_account = UserCredit.objects.select_for_update().get(id=user_id)
          
          if user_account.available_credit < amount:
              raise InsufficientCreditError("Credit limit exceeded.")
              
          # Deduct credit safely via locked row
          user_account.available_credit -= amount
          user_account.save()
          
          # Record Ledger Entry
          Ledger.objects.create(
              user_id=user_id,
              merchant_id=merchant_id,
              amount=amount,
              transaction_type='DEBIT_BNPL',
              status='CONFIRMED'
          )
          
          return {"status": "success", "approved_amount": amount}
  ```

### <img src="https://raw.githubusercontent.com/Tarikul-Islam-Anik/Animated-Fluent-Emojis/master/Emojis/Travel%20and%20places/Milky%20Way.png" width="30"/> Project 3: ASTRODATE вҖ“ AI Matchmaking

<details>
  <summary><b>рҹҢҢ Expand AI Recommendation Engine specifics</b></summary>

  **Executive Summary:**
  Astrodate merges ancient astrological heuristics with modern machine learning. It acts as an intelligent dating platform that computes compatibility not just on user preferences, but on complex planetary alignments and behavioral cluster analysis.

  **Machine Learning Architecture Breakdown:**
  * **Data Pipeline:** Ingests user birth data (lat, long, time, date) and calls ephemeris APIs to generate a 12-house planetary matrix.
  * **AI Recommendation Engine:** A hybrid filtering system. First pass is collaborative filtering (who likes who). Second pass is a custom Scikit-Learn Content-Based filter utilizing Euclidean distance on normalized astrological traits.
  * **Model Deployment:** Wrapped the Python ML engine in a lightweight Flask container.

  **Key Engineering Challenges Solved:**
  1. *High-Dimensional Matrix Computation:* Calculating similarity scores across thousands of users concurrently required vectorizing operations using NumPy rather than standard loops, reducing calculation time by 90%.
  2. *Cold Start Problem:* Solved standard dating algorithm cold-starts by allowing the astrological heuristics to provide immediate match scoring even before a user had "swiped" on anyone.

  **Sample Implementation Snippet (Recommendation Engine Logic):**
  ```python
  # Python: Vectorized Compatibility Scoring using Cosine Similarity
  import numpy as np
  from sklearn.metrics.pairwise import cosine_similarity

  class AstroMatchEngine:
      def __init__(self, active_user_vector, database_matrix):
          # active_user_vector: 1D array of normalized astro-traits
          # database_matrix: 2D array of all potential matches
          self.user = np.array(active_user_vector).reshape(1, -1)
          self.candidates = np.array(database_matrix)
          
      def compute_scores(self):
          """Compute similarity across all network candidates instantly"""
          distances = cosine_similarity(self.user, self.candidates)
          
          # Flatten array and sort indices by highest similarity
          scores = distances.flatten()
          best_match_indices = np.argsort(scores)[::-1] 
          
          return [(index, scores[index]) for index in best_match_indices[:10]]
  
  # Execution is sub-millisecond for thousands of rows thanks to vectorized math.
  ```
</details>
